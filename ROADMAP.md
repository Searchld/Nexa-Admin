# Roadmap

本路线图用于说明 Art Design Pro RuoYi 后续演进方向。优先级会根据 Issue、Discussions 和实际使用反馈调整。

## 近期计划

- [ ] 增加代码生成器演示 GIF，展示导入表、选择 Art 模板、预览代码和下载生成结果。
- [x] 补充部署文档，覆盖本地开发、生产部署、Nginx 代理、Redis、MySQL 和静态资源部署。
- [x] 增加 GitHub Actions，自动执行后端 Maven 测试、前端类型检查和空白错误检查。
- [ ] 完善 Art Bot 使用文档，补充 OpenAI 兼容服务配置示例、常见错误和安全建议。
- [ ] 补充系统菜单、权限标识、SQL 迁移顺序和升级注意事项。

## 中期计划

- [ ] 继续补齐 RuoYi 生态常用页面的 Art 风格适配。
- [ ] 优化代码生成模板，提升字段类型推断、搜索项布局、树表交互和主子表体验。
- [ ] 增加更多 Art 业务组件示例，方便二次开发复用。
- [ ] 增加更完整的审计治理，区分普通查询、高频隐私操作和敏感读写操作。
- [ ] 增加可选的 Docker Compose 开发环境。

## 长期计划

- [ ] 形成稳定的版本升级指南和迁移规范。
- [ ] 建立组件、页面和模板贡献规范。
- [ ] 支持更多 OpenAI 兼容模型服务的配置示例。
- [ ] 根据社区反馈拆分更多可复用前端业务组件。

## 不在首版范围

- 不重写 RuoYi RBAC、JWT、菜单权限和数据权限。
- 不把项目改成纯前端静态权限模式。
- 不新增与 RuoYi 原接口重复的接口体系。
- 不默认引入厂商专属 AI 协议，首版 Art Bot 聚焦 OpenAI Chat Completions 兼容协议。
